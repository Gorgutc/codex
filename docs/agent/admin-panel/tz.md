# ТЗ: Админ-панель Codex Studio

Источник: владелец репозитория (<axximus@gmail.com>), сессия 2026-06-10.
Статус: утверждено владельцем, решения зафиксированы.
План итераций: см. `handoff.md` в этой папке.

## Цель (goal)

У сайта <https://codex.promo/> есть внешняя оболочка, но владельцу-администратору нужна
удобная и понятная административная панель, дающая полный контроль над контентом сайта
без участия разработчика.

Через админ-панель администратор должен уметь:

- менять теги и мета-теги;
- менять фотографии и иллюстрации;
- менять и заливать видеоматериалы;
- переписывать весь текстовый контент (RU и EN);
- менять блоки местами внутри кейсов;
- включать и выключать отдельные кейсы или целые категории.

## Главное ограничение

Все существующие визуальные и технологические решения сайта должны быть сохранены:
GSAP-анимации, seeded shuffle раскладки кейсов, 3D-viewer, i18n RU/EN, SEO-разметка,
кэширование Netlify. Цель — контроль над контентом, а не редизайн.

## Сопутствующие требования владельца

1. Проверить агентов, скиллы, хуки и инструкции репозитория; восстановить обвязку
   Claude Code с полным паритетом к Codex-обвязке: идентичные дублирующиеся скиллы,
   чтобы обе системы работали в разных сессиях независимо друг от друга.
2. Держать ТЗ перед собой (этот документ).
3. Вести продуктовый ресерч удобных админок и применять решения (см. `research.md`).
4. Использовать для ревью решений и кода Codex: `/codex:review` и
   `/codex:adversarial-review` (плагин `openai/codex-plugin-cc`).
5. Все решения фиксировать в handoff (см. `handoff.md`), чтобы работу можно было
   продолжить из любой сессии.

## Утверждённые решения (2026-06-10)

| Вопрос | Решение |
| --- | --- |
| Тип админки | Кастомная vanilla-админка `/admin` в этом же репозитории; Sveltia CMS — задокументированный план Б |
| Видео | Короткие лупы `.webm` (до ~20–30 МБ) — в репозиторий; тяжёлые ролики — вручную на Vimeo, в админке поле URL/ID (`vimeoId` уже поддерживается сайтом) |
| Публикация | Автосохранение черновика → превью «как будет» → явная кнопка «Опубликовать» (один коммит → деплой Netlify ~2–3 мин); откат в один клик через историю |
| Вход | GitHub OAuth через небольшую Netlify Function; доступ = write-доступ к `Gorgutc/codex` |

## Архитектура контента (фундамент)

Контент извлекается в `content/*.json`. Node-генератор `scripts/generate-content.mjs`
детерминированно собирает из них продакшн-файлы: `js/cards-data.js`, `js/fa-data.js`,
`js/i18n-data.js` и регионы `index.html` / `free-assets.html` между маркерами
`<!-- CODEX:GEN ... -->`. GitHub Action коммитит сгенерированное обратно в репозиторий,
поэтому инвариант «репозиторий = задеплоенный сайт» сохраняется, Netlify остаётся
статическим хостом без build-шага. Runtime-загрузка JSON отвергнута (SEO, LCP,
verify-frozen). Seeded shuffle (`mulberry32` в `buildItems()`) не меняется — раскладка
остаётся прежней.

## Открытые вопросы (не блокируют)

- Русские тексты кейсов сейчас равны английским (плейсхолдеры): владелец заполнит их
  через админку после итерации D.
- ZIP-файлы free-assets (до 340 МБ по описаниям) превышают лимит GitHub 100 МБ на файл —
  управление ими через админку отложено (потребует внешнего хранилища).
- `/codex:setup` требует авторизации OpenAI (ChatGPT-логин или API-ключ) — шаг владельца.
