### Детали, которые берём

- Scroll-driven анимации через GSAP ScrollTrigger
- Типографические заголовки (display-шрифт на тёмном фоне)
- Full-bleed секции
- Плавные переходы (fade + translate, без резких скачков)
- Визуальный ритм (разная высота секций / блоков)
- Custom cursor с тонкой реакцией на hover (опционально, только pointer:fine)
- Атмосферные слои (film-grain, vignette) для «плёночного» ощущения
- Технические UI-паттерны (counter `n / N`, COPY LINK, tabs 2D/3D/Blueprints) — индустриальная эстетика, не SaaS
- Вертикальная галерея с lift-on-scroll easing для item-ов кейса
- Sidebar + content layout (как в технических документациях / IDE) — намеренно НЕ хедер-героy-сетка

## 🎯 Codex-фильтр (применять после анализа референса)

Любой элемент референса прогонять через:

1. **Палитра** — заменить на `--color-bg #212121` + `--color-primary #327AAE` (или `--color-secondary #2E8B8F`)
2. **Типография** — Clash Display (от 24px) + General Sans (всё ниже)
3. **Скруглённость** — только из шкалы `--radius-*`
4. **Анимации** — только `opacity` + `transform`, easing из `--ease-*`
5. **Decorative AI-паттерны** — удалить полностью (не «ослабить», не «адаптировать»)

## 🔍 Проверочный вопрос

Перед добавлением любого элемента:
> «Это добавляет смысл или просто заполняет пространство?»

Если ответ «заполняет пространство» — убрать.

## 📌 Архитектурные особенности проекта (учитывать при адаптации)

- `<body data-theme="dark">` хардкод → нельзя использовать `prefers-color-scheme` для авто-переключения; theme-color только single-tag
- Sidebar всегда виден на desktop (340px), на mobile становится bottom-overlay
- Custom cursor — только `pointer: fine` + не reduced-motion
- 3D-вьювер через `<model-viewer>` — lazy-load
- `model-data.js` 1.1 MB inline GLB — lazy-load, не делать eager
- Tag-cards на FA имеют двойной класс `tag-card work-card` → любое правило для `.work-card` в анимациях должно использовать `:not(.tag-card)` фильтр

При адаптации референса — учитывать эти константы. Не предлагать паттерны, которые их нарушают.
